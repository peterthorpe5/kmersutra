KmerSutra v0.26.0 release notes
================================

Main change
-----------
This release makes hierarchical cascade screening substantially more scalable.
The older hierarchical implementation screened each gate panel and each activated
module panel separately, which meant repeatedly reading large FASTQ/FASTA inputs.

v0.26.0 now uses a two-pass hierarchical screen:

1. Load all gate panels into one combined gate index and screen the input once.
2. Activate modules from the combined gate evidence.
3. Load all activated module panels into one combined module index and screen the
   input once more.

This reduces true hierarchical screening to at most two full input passes,
regardless of the number of modules, while preserving the conservative evidence
and lineage-aware reporting logic.

Compatibility
-------------
Flat single-panel screening still works. Hierarchical remains the default screen
mode, with single-panel fallback when no module manifest is supplied.

Testing
-------
The unittest suite passed after adding focused tests for combined gate indices,
module membership tracking and two-pass hierarchical screening.
