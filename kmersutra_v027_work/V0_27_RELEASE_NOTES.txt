KmerSutra v0.27.0
==================

Main change
-----------
KmerSutra panel builds now automatically export hierarchical screening inputs
from the final flat diagnostic panel:

- hierarchical_modules/kmersutra_module_manifest.tsv
- hierarchical_modules/gates/*.gate.tsv.gz
- hierarchical_modules/modules/*.module.tsv.gz
- hierarchical_modules/module_export_summary.tsv

This makes global-candidate databases immediately usable by the v0.26 two-pass
hierarchical screening engine. Flat panel output is still written as before.

Design notes
------------
The automatic exporter derives conservative gate/detail modules from the
already validated final panel. It does not change marker evidence ranks. Broad
rank records are used as gate markers where available. Species-marker gate
fallback is enabled by default for genera with no broad-rank gate evidence so
that those modules can still be activated for known-species signals.

New build options
-----------------
--write_module_manifest / --no_write_module_manifest
--module_manifest_dir
--module_gate_ranks
--module_min_gate_unique_kmers
--module_min_gate_positive_sequences
--module_min_gate_k_values
--module_min_gate_best_k
--module_max_gate_records_per_k
--no_species_gate_fallback

Testing
-------
The unittest suite passed after adding tests for automatic module export and
loadable module manifests.
